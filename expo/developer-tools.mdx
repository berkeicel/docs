## Debug Mode

Enable development environment:

```typescript
await RampKit.init({
  appId: "YOUR_APP_ID",
  environment: "development",
});
```

This enables:
- Verbose logging with `[RampKit]` prefix
- Additional debug information
- Easier troubleshooting

## Inspect Device Context

```typescript
import { collectDeviceInfo, buildRampKitContext } from "rampkit-expo-dev";

async function debugContext() {
  const deviceInfo = await collectDeviceInfo();
  console.log("Device Info:", JSON.stringify(deviceInfo, null, 2));
  
  const context = buildRampKitContext(deviceInfo);
  console.log("Template Context:", JSON.stringify(context, null, 2));
}
```

## Verify Onboarding Data

```typescript
const data = RampKit.getOnboardingData();
console.log("Screens:", data?.screens?.length);
console.log("Screen IDs:", data?.screens?.map(s => s.id));
```

## Test Native APIs

```typescript
import { Haptics, StoreReview, Notifications } from "rampkit-expo-dev";

// Test haptics
await Haptics.impactAsync("medium");

// Test store review
const available = await StoreReview.isAvailableAsync();
console.log("Review available:", available);

// Test notifications
const status = await Notifications.getPermissionsAsync();
console.log("Notification status:", status);
```

## Console Filtering

Filter React Native logs:
- Search: `[RampKit]`
- Shows: Init, events, errors

## Best Practices

<Tip>
- Always test on real devices for haptics and purchases
- Use development environment during development
- Check logs when debugging issues
- Test with different device contexts
</Tip>
